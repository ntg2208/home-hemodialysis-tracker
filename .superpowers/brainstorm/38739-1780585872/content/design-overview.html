AI Command Control — Full Design

Gemini tool use → AppCommand bus → screens react. Read-only auto, writes fill-and-preview.

How it works end-to-end

1. You: "show my Hb for the last 3 months"
chat sends prompt + tool declarations to Gemini
2. Gemini responds with tool call: filter_blood_tests(marker:"haemoglobin", months:3)
GeminiChatResponder dispatches AppCommand via callback
3. CommandBus receives: FilterBloodTests(marker:"haemoglobin", months:3)
NavigateTo + BloodTests screen listens to bus
4. Router goes to /blood-tests • filter auto-applies • chart shows Hb/3mo
Gemini gets tool result → generates reply text
5. Chat reply: "Here's your Hb trend for the last 3 months. Latest: 112 g/L (↑ from 108)."

Command vocabulary (all tools Gemini can call)

Tool name Parameters Execution Example prompt
navigate_to route: string auto "go to inventory"
filter_blood_tests marker?, phase?, months?, tab? auto "show Hb last 3 months"
filter_fitness type?, days? auto "show my steps this week"
prefill_pre_treatment weight?, bp_sys?, bp_dia?, pulse?, uf_goal?, uf_rate? fill + wait "start session, weight 72.4, BP 140/85"
prefill_reading bp_sys?, bp_dia?, pulse?, blood_flow?, vp?, ap? fill + wait "add reading, BP 130/80, flow 350"
prefill_post_treatment weight?, bp_sys?, bp_dia?, pulse?, total_uf? fill + wait "end session, weight 70.1, BP 122/76"
