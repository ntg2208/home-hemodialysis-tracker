How should AI control work?

The app already has Gemini chat for Q&A. Which direction do you want to take AI control?

💬 Chat
You: start a session, pre-weight 72.4, BP 140/85
AI: Opening pre-treatment form… ✓
AI: Filled weight=72.4, BP=140/85 ✓
You: now show Hb for last 3 months
AI: Navigating to Blood Tests → Hb, 3-month view ✓

A — Chat drives the app

Extend the existing Gemini chat with "tool use": the AI can navigate screens, pre-fill forms, and apply filters — all via conversation inside the app. You talk to it, it acts.

⌨️ Command palette ⌘K
› start session pre-weight 72.4…
▸ Start session — Pre-treatment form
Show blood tests — Hb, 3 months
Check inventory stock

B — Command palette

A global search/command overlay (tap or swipe to open). Type a command, AI resolves it to an action. Faster than chat — no back-and-forth, just instant execution.

External AI (Claude Code, MCP, automation)
→ POST /api/app-command
{
  "command": "navigate",
  "screen": "blood-tests",
  "params": {"marker":"Hb","months":3}
}
← 200 OK — app updated

C — External AI control API

The app exposes a command API (or MCP server). Claude Code, automated workflows, or any external AI agent can send commands to drive the app. No in-app chat needed.

💬 Chat → drives app (in-app)
+ tool use for navigation & forms
AND
🔌 External command API
MCP server or /api/command endpoint

D — Both

In-app chat with tool use for day-to-day use, plus an external API for automation and Claude Code integration. Same underlying command layer powering both.
