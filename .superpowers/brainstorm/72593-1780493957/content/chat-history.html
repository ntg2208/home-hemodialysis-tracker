Firestore Schema

Collectionchat_conversations/{conversation_id}
Fieldsid: string, title: string (first message, truncated), created_at: timestamp, updated_at: timestamp, messages: [{role: 'user'|'assistant', text: string, timestamp}]
Client accessDirect Firestore from Flutter (same pattern as treatment sessions). No Cloud Run proxy.
Firestore rulesNew chat_conversations collection, same UID lock as treatment_* collections
LifecycleAuto-saved on each assistant reply. "New chat" creates a fresh conversation. "Clear all history" deletes all docs.
