Chat Architecture — Fully On-Device

Flutter assembles context, calls Gemini AI Studio directly. Cloud Run not involved.

Request Flow

Flutter ChatController.send(message)
1. Build patient summary from local caches
2. Read API key from flutter_secure_storage
3. POST to generativeai.googleapis.com (SSE stream)
BtStore
blood tests cache
Firestore
sessions + inventory
CacheStore
fitness summary
Secure Storage
AI Studio key
↓
Gemini AI Studio
https://generativeai.googleapis.com/v1beta/models/gemini-2.5-flash:streamGenerateContent
Auth: x-goog-api-key header
Response: SSE stream with content chunks
↓
ChatSheet rebuilds
SSE chunks → words append in real-time to markdown bubble
No Cloud Run, no cold starts. Only online dependency: Gemini API.

What Stays vs Goes

API keyStored in flutter_secure_storage alongside existing main key. Entered once in Settings.
Context assemblyNew ChatContextBuilder class in Flutter — reads from BtStore / Firestore / CacheStore, returns a formatted string.
LLM callDirect HTTP to Gemini REST API. SSE stream parsed in Dart. No proxy.
Cloud RunNot involved. /api/chat placeholder stays as-is or can be removed.
OfflineContext works offline. LLM call fails gracefully — "AI requires internet" message.
