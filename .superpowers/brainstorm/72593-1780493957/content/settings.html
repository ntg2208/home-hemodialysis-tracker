Behaviour Rules

Toggle OFFChat FAB still visible. Sheet opens with a "AI is disabled — enable in Settings" message and a link to Settings. No API calls possible.
Toggle ON + no keyChat sheet opens but send returns "API key not set — add one in Settings."
Toggle ON + key setFull chat works. Key read from flutter_secure_storage on each send.
Clear keyConfirmation dialog: "Clear AI Studio key?" → Yes: remove from storage. Toggle stays ON, falls to "no key" state.
Toggle OFF while key setKey stays stored — re-enabling restores chat immediately without re-entering.
Paste from clipboardPaste button next to key field. Reads clipboard, validates it looks like an API key, auto-fills.

API Key Storage

// flutter_secure_storage keys (new additions bolded)
'main_api_key' → existing (auth for Cloud Run /api/*)
'ai_enabled' → NEW 'true' | 'false'
'ai_studio_key' → NEW Gemini API key string

All three are read on app start to configure the chat provider.
Key is never sent to Cloud Run — stays on-device.
